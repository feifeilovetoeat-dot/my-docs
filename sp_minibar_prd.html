3.需求背景（必须）
3.1 背景

SP 场景下，未登录用户触发 SP_Result_CB_Popup，当前 88.6% 的用户选择直接关闭弹窗，之后没有任何登录引导持续触达用户，转化机会彻底丢失。

用户关闭弹窗不等于"永远不登录"，更可能是"现在不想被打断"。如果能在用户继续购物的过程中保持低干扰的登录入口，在用户准备结账或有空余注意力时触发登录，可以有效延长转化窗口。

本需求在用户关闭弹窗后，将登录引导收起为页面底部的一条 Persistent Mini-Bar，不拦截操作，用户可随时点击或关闭。

5.终端要求（必须）
项目说明
平台 Chrome / Edge / Firefox / Safari Extension
触发条件 用户未登录 + 已触发 SP_Result_CB_Popup + 用户点击弹窗关闭按钮（exit_button）
不触发条件 用户已点击登录按钮（sign_in_button）后不展示 Mini-Bar
用户已登录不展示
用户在当前页面 session 内已主动关闭过 Mini-Bar 不再出现
展示位置 插件弹窗底部固定悬浮条，不遮挡页面主体内容，z-index 需高于普通页面元素
展示时长 持续展示，直到用户主动关闭 或 用户完成登录 或 离开当前 tab
点击行为 点击 Mini-Bar 主体区域 → 触发登录流程（与原弹窗登录按钮行为一致）
点击 ✕ → Mini-Bar 消失，当前 session 不再出现
降级策略 无法获取返现金额时，隐藏金额展示通用文案：Cash Back pending · Log in to claim
多语言 跟随现有插件语言体系，需提供 i18n key
6.需求明细（必须）
6.1 Mini-Bar 展示逻辑
功能点功能说明
触发入口 监听 SP_Result_CB_Popup 的关闭事件（exit_button click）
弹窗关闭动画结束后，Mini-Bar 从底部滑入（动画时长建议 200ms）
内容组成 ① 商家 Logo（小图标，16×16px）
② 金额文案：💰 $X.XX Cash Back pending
③ 行动文案：Log in to claim →
④ 关闭按钮：✕
金额数据来源 复用 SP_Result_CB_Popup 已获取的当前激活 CB 预估金额
无需额外接口请求
关闭逻辑 点击 ✕ → 隐藏 Mini-Bar，写入 session storage 标记 minibar_closed=true
同一 tab session 内不再展示
新 tab 或刷新页面后重置（不做跨 session 持久化）
登录成功后 用户完成登录后，Mini-Bar 自动消失
6.2 交互细节
场景行为
用户点击 Mini-Bar 主体 直接唤起登录流程（弹出登录 popup 或跳转登录页，与原按钮行为一致）
用户滚动页面 Mini-Bar 固定在底部，不随滚动消失
用户切换 Tab Mini-Bar 随 Tab 隐藏/恢复，不跨 Tab 展示
用户关闭 Tab Mini-Bar 随 Tab 销毁，session 标记清除
再次进入同一商家页 若已关闭 Mini-Bar（session 内），不重新出现
6.3 埋点要求
事件名触发时机需记录字段
sp_minibar_show Mini-Bar 首次出现 client_id, domain, cb_amount, has_amount(是否有金额)
sp_minibar_click 用户点击 Mini-Bar 主体触发登录 client_id, domain, cb_amount
sp_minibar_close 用户点击 ✕ 关闭 client_id, domain, time_visible_ms(展示时长)
sp_minibar_login_success 通过 Mini-Bar 点击后完成登录 client_id, domain
💡 埋点说明：time_visible_ms 记录 Mini-Bar 从出现到关闭的毫秒数，用于分析用户在什么时间节点选择关闭，帮助后续判断是否需要调整出现时机。
